# Prozesse

## Feature Development
1. Issue im Repo erstellen
2. Branch: `feature/issue-123`
3. Commit → Push → PR
4. Review → Merge zu `main`

## Automatisierung (n8n)
- Trigger: Webhook, Schedule, Manual
- Integrations: GitHub, Google, Firebase
- Logs: n8n UI

## Communication
- Async-first (Slack für Urgent)
- Status im Issue tracken
- Weekly Sync: Montag 10:00
